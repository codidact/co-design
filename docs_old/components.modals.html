Modals

Modals are popup views that interrupt the normal user flow to maximize their attention.

Only CSS! Co-Design is a CSS-only framework and therefore doesn't come with the JavaScript neccessary to display and hide a modal. You'll need to write JavaScript on your own or use an existing library, in order to be able to effectively use modals.

Basic structure

A basic modal consists of the following:

  • A modal wrapper (.modal), with
    • A modal container (.modal--container), that contains, in this order:
      • A modal header (.modal--header), that contains, in this order:
        • A modal close button (.button.is-close-button.modal--header-button)
        • A (text) title for the modal
      • A modal body (.modal--body)
      • A modal footer (.modal--footer; optional)

The modal body and footer can be filled with any HTML content, however the footer is fixed to the bottom on mobile and should therefore be used only for few elements with few height.

You need to give a modal the class .is-active, in order for it to be shown.

Example
<div class="modal">
  <div class="modal--container">
    <div class="modal--header">
      <button class="button is-close-button modal--header-button">×</button>
      An example modal
    </div>
    <div class="modal--body">
      <p>...</p>
    </div>
    <div class="modal--footer">
      <button class="button is-filled">Approve</button>
      <button class="button">Reject</button>
    </div>
  </div>
</div>

Modal with backdrop

When a modal needs your full attention, so that nothing else may be done while it is active, it's possible to add a "backdrop", by giving the .modal the class .is-with-backdrop.

Example
<div class="modal is-with-backdrop">
  <div class="modal--container">
    <div class="modal--header">
      <button class="button is-close-button modal--header-button">×</button>
      An example modal
    </div>
    <div class="modal--body">
      <p>...</p>
    </div>
    <div class="modal--footer">
      <button class="button is-filled">Approve</button>
      <button class="button">Reject</button>
    </div>
  </div>
</div>

Danger modal

Sometimes a modal shows dangerous things, or dangerous things might happen if you confirm the main action. In these cases, it's appropriate to give the .modal the class .is-danger.

Of course the actions would be dangerous, too, so you should also add the class .is-danger to buttons inside the modal

Example
<div class="modal is-danger">
  <div class="modal--container">
    <div class="modal--header">
      <button class="button is-close-button modal--header-button">×</button>
      An example modal
    </div>
    <div class="modal--body">
      <p>...</p>
    </div>
    <div class="modal--footer">
      <button class="button is-danger is-filled">Approve</button>
      <button class="button is-danger">Reject</button>
    </div>
  </div>
</div>
